## 0. TL;DR

1. **API hh.ru для соискателей закрыт 15 декабря 2025.** Это бьёт по продукту, не по SEO. Подход «отклик через API» мёртв; выжившие — на headless + OTP + серверных сессиях против WAF/TLS-fingerprint/поведенческой биометрики.
2. **Ниша не пустая — переполнена.** ~12 коммерческих конкурентов по «автоотклик hh», у части готовые блог-кластеры. Исходный ресерч утверждал обратное («конкуренция минимальна, можно застолбить нишу») — это неверно.
3. **Не воевать за head-термин «автоотклик hh».** Коммодити, race-to-bottom (399₽), последнее место из 12, и это ровно тот «массовый отклик», который hh давит и Хабр хоронит.
4. **Реальный дифференциатор Otclick — пост-откликовый слой:** агент отвечает рекрутёрам (human-in-loop + escalate-to-human) и ведёт todo с задачами от рекрутёра (формы, выбор времени созвона). Никто из 12 это не продаёт. Здесь SEO-территория без боя.

## 1. БОМБА 1 — Публичный API hh.ru закрыт (15 дек 2025)

Подтверждено тремя свежими статьями Хабра (окт–дек 2025):

- Старый путь «отклик через публичный API» **больше не работает**.
- Защита hh сейчас: **WAF + анализ TLS-fingerprint + поведенческая биометрика + shadow-ban**.
- Прямая цитата: *«Массовый автоотклик умер полностью. Инструменты, обещавшие 1000 откликов/день, либо превратились в ничто, либо стали троянами для аккаунта»*.
- Выжившие сервисы перешли на **headless-браузер + OTP-авторизацию + защищённые серверные сессии**.

**Влияние на проект:** текущая архитектура (`ApiClient`, OAuth, `/resumes/mine`, apply через API в `CLAUDE.md`) опирается на закрытый API. Вектор на Playwright/headless — правильный, но теперь против вас работает анти-фрод hh.

**Прямой кейс для чтения:** Хабр 983318 — «Как мы учили Headless Chrome притворяться живым человеком (RPA против Anti-Fraud)».

> Архитектурная заметка по нашему стеку: Playwright используется только для auth+captcha
> (экономия на инференсе), остальные действия — прямыми запросами. Риск: прямой
> не-браузерный трафик после закрытия API легче палится по TLS-fingerprint и поведению.
> Проверить, что прямые запросы воспроизводят браузерный fingerprint сессии, поднятой Playwright.

## 4. Позиционирование — не голова воронки, а её хвост

12 конкурентов продают одно: **«200 откликов/день»**. Это коммодити, опасно (анти-фрод hh) и место №12 для нового домена.

**Уникальный слой Otclick — после отклика:**
- агент отвечает рекрутёрам (human-in-loop + tool `escalate to human`);
- todo-оркестратор с задачами от рекрутёра: заполнить гугл/яндекс-форму, выбрать время созвона, написать в телеграм.

> Тестовые задания вместо пользователя делать **не** планируем — из позиционирования и
> keyword-плана исключено.

Никто из 12 не продаёт пост-откликовый слой как фичу — все бросают юзера на «отклик отправлен». **Otclick ведёт до оффера.** Это и УТП, и незанятая SEO-территория.

### Что эти числа говорят

1. **Исходный Semrush-ресерч сильно занижал спрос.** Semrush (Google): «сопроводительное письмо 14 800». Яндекс: **76 151** — в 5× больше. «сопроводительное письмо hh» Semrush=720, Яндекс=2 245 (3×). Подтверждает: планировать по Semrush в РФ нельзя, Яндекс — основной рынок.

2. **Прямой продуктовый спрос реально крошечный.** «автоотклик hh» = 478, «автоотклик» = 725, «бот для откликов hh» = 31. Воевать за head-термин (где сидят 12 конкурентов) ради 478/мес — бессмысленно. Исходный ресерч тут оказался прав по сути, хоть и по неверной причине.

3. **KZ через Яндекс — пустыня.** Почти всё 1–22 или ERR. Казахстан = Google-доминантный + меньше населения. **SEO под hh.kz через Яндекс смысла не имеет.** Для KZ — отдельная стратегия на Google, оценивать через другой инструмент.

4. **«Трекинг откликов» — мёртвый кластер.** трекер вакансий 38, таблица откликов 14, как вести поиск работы 43. Из раздела 5 этот кластер **убрать** — спроса нет.

## 5c. Go-to-market: как привлечь максимум релевантных пользователей

Главный принцип из чисел: прямого спроса на «автоотклик» почти нет (478/мес), а на «сопроводительное письмо» — 76 151/мес. Поэтому вход в воронку — не оффер «бот-автоотклик», а **бесплатный генератор сопроводок** (он уже есть в коде, `cover_letter.py`). Автоотклик + агент-ответчик = апселл.

### Воронка по приоритету

**1. Лид-магнит — бесплатный генератор сопроводительных писем.**
Ловит самый большой трафик ниши (76k) и конвертит инфо-интент в аккаунты:
- Лендинг `/soprovoditelnoe-pismo`: вставь вакансию + резюме → AI-письмо за секунды.
- Первые 3 письма бесплатно без регистрации (модель quick-offer).
- 4-е → регистрация → апселл: «хочешь, сами откликаемся и отвечаем рекрутёрам?».

**2. SEO-контент под живые подзапросы** (не под «автоотклик» — там 478 и 12 конкурентов). По статье на запрос, CTA = бесплатный генератор:
- сопроводительное письмо на вакансию hh (480), что писать в сопроводительном на hh (299), как написать сопроводительное на hh (164).
- AI на опережение: нейросеть для резюме (765), ии агент для поиска работы (30 — буквально продукт).

**3. Каналы где ЦА реально сидит (не только поиск):**
- **Telegram** — там живут конкуренты (autootklik, MrJobHunter). Канал «вакансии + лайфхаки» + бот-генератор сопроводок как точка входа.
- **vc.ru / Habr / DTF** — горячая тема 2026 «автоотклик после закрытия API hh». Кейс «как держим автоотклик живым после закрытия API» = тех-аудитория + SEO.
- Профильные чаты, Пикабу, job-сообщества — органика через пользу.

**4. Месседж = дифференциатор, не «200 откликов»:**
> «Не просто откликаемся — отвечаем рекрутёрам и ведём до оффера. Ты приходишь только на собеседование.»
Незанятая территория: «ответы рекрутёрам» (что отвечать работодателю — 2 524), ни один сервис не таргетит.

**5. Product-led петля.** Футер «✨ создано в Otclick» на бесплатных письмах (опционально) — вирусный вход.

### Чего НЕ делать
- ❌ Бюджет/лендинги под «автоотклик hh» — 478/мес, 12 конкурентов, race-to-bottom 399₽.
- ❌ SEO под hh.kz через Яндекс — спроса нет (1–22/ERR). Казахстан позже, через Google.
- ❌ Обещать «1000 откликов/день» — это то, что hh банит, а Хабр хоронит; убивает доверие.

**Итог:** бесплатный генератор сопроводок = верх воронки (76k спроса, дёшево и массово), автоотклик + агент-рекрутёр = монетизация (уникальный продукт на выходе).

## 7. Открытые вопросы перед SEO-инвестициями

1. **Анти-фрод hh** — как держим сессии живыми после закрытия API (прямые запросы vs fingerprint). До этого SEO=привлечение юзеров на нестабильный продукт.
2. **Юридический риск** — публичная воронка на бренде делает продукт видимым для самого hh (cease-and-desist / блок). Решить позиционирование («ассистент соискателя», не «обход hh»).
3. **HH.kz** — Semrush и Вордстат по KZ разрежены; если Казахстан целевой — отдельная оценка спроса нужна.
4. **Качество авто-ответов рекрутёрам** — плохой ответ жжёт оффер сильнее лишнего отклика; human-in-loop + escalate уже есть, держать preview перед отправкой.
