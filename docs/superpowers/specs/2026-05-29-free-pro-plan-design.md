# Дизайн: Free / Pro план Otclick

> Дата: 2026-05-29. Статус: согласован, ждёт ревью спеки.
> Контекст: `PRICING.md`, `SEO-RESEARCH-VERIFIED.md`.

## Проблема

Сопроводки сейчас генерятся **только внутри autopilot-флоу** за авторизацией
(`backend/app/services/cover_letter.py`, вызывается из `apply.apply_one`). Публично —
только лендинг (`frontend/src/app/page.tsx`). Отдельной страницы-генератора нет.

Нужно определить границу free / pro и понять, что для этого построить.

## Решающий принцип

Дифференциатор Otclick — **агент отвечает рекрутёрам + ведёт задачи до оффера** —
проявляется только ПОСЛЕ ответа рекрутёра (через 3-7 дней после отклика). Поэтому:

- короткий триал (3 дня) **не успевает показать** главную ценность → плохая модель;
- лимит откликов на free → free-юзеры откликаются под нашим риском (анти-фрод, саппорт,
  инференс на неплатящих), а moat всё равно проявится позже лимита → плохая модель;
- **free должен давать мгновенную ценность** (письма/вакансии видны сразу), а исполнение
  + после-откликовый слой — за деньги, где moat и живёт.

Решение: **free = «покажем, что сделаем» (подготовка), pro = «делаем + ведём до оффера»
(исполнение).** Paywall проходит ровно по линии нашего moat.

## Граница Free / Pro

### FREE (без карты, бессрочно)
- Подключение hh-аккаунта, синхронизация резюме.
- Подбор вакансий по фильтрам — юзер видит очередь «откликнёмся на это» (просмотр).
- Ручная генерация сопроводок: выбрал вакансию → «сгенерировать письмо» → просмотр/копирование.
- **Лимит: 5 генераций писем / день** (на все вакансии письмо не нужно; cap на инференс).
- НЕТ: авто-отклика (worker не стартует), ответов рекрутёрам, todo-оркестратора,
  решения тестов, автоподнятия резюме.

### PRO
- Всё из free +
- авто-отклик (worker): **30 откликов/день** (месяц) / **25/день** (спринт);
- **ответы рекрутёрам** (агент, human-in-loop) — moat;
- **todo-оркестратор** задач от рекрутёра — moat;
- решение тестов работодателей (form filler);
- автоподнятие резюме.

### Тарифы (из PRICING.md)
- Спринт 7 дней — 1 990 ₽ (разово), 25 откликов/день.
- Месяц — 3 900 ₽/мес (авто-продление, отмена в 1 клик), 30 откликов/день.

## Что построить

**Новая UX-поверхность: ручной режим генератора в app.** Сейчас письмо генерится только
как часть worker-флоу; для free нужен путь «вакансия → кнопка → письмо → копировать» БЕЗ
запуска worker. Поверх существующего `cover_letter.py` (движок переиспользуем).

Компоненты (предварительно, уточнить на этапе плана):
- фронт: страница/секция со списком подобранных вакансий + кнопка генерации + просмотр письма;
- бэк: эндпоинт «сгенерировать письмо для вакансии X по резюме Y» вне apply-пайплайна;
- gating: `plan.has_access` уже различает план; добавить проверку «free → только генерация,
  pro → worker» + дневной лимит генераций для free (по образцу `worker/limiter.py`).

## Гейтинг (опираемся на существующее)
- `services/plan.py` (`has_access`/`check_access`) уже гейтит старт worker — pro-фичи вешаем сюда.
- Дневной лимит генераций free — отдельный счётчик (паттерн `apply_counters` / `limiter.py`).

## Привлечение (следствие решения)
Без публичной standalone-страницы free НЕ ловит SEO-спрос 76k (он на «простой сайт-генератор
без логина»). Значит верх воронки — реклама / TG / контент со ссылкой на регистрацию, а не
органика из поиска по «сопроводительное письмо». Зафиксировано осознанно.

## Не делаем (YAGNI)
- Публичную страницу-генератор без логина (решено: только внутри app).
- 3-дневный pro-триал — не нужен.
- Тизер moat для free («что бы ответили рекрутёру») — не нужен.
- Lifetime-тариф (как sofi 28800 ₽) — не в MVP, искажает LTV.
- «200 откликов/день» — анти-фрод убивает, не продаём.
